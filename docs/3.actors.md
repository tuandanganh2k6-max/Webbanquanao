# 👥 Actors & User Roles

## 1. Actor Overview

An **actor** is any entity that interacts with the system. Webbanquanao has four primary actors with different levels of access and responsibilities.

---

## 2. Actor Definitions

### 2.1 Guest (Unregistered User)

**Definition:** A person who visits the Webbanquanao website without creating an account or logging in.

#### Characteristics
- No authentication required
- Limited access to read-only features
- Session-based experience (no data persistence)
- Can browse but cannot make purchases
- Cannot view personal information or orders

#### Responsibilities
- Browse and view products
- Use search and filter functionality
- Read product details and reviews (if available)
- Register to become a Customer

#### Use Cases
- UC01: View Product Catalog
- UC02: Search Products
- UC03: Filter Products
- UC04: Register Account
- UC05: View Product Details

#### Primary Goals
- Explore available products
- Make purchase decisions
- Learn about the store

---

### 2.2 Customer (Registered User)

**Definition:** A person who has created an account and is logged into the system with basic user permissions.

#### Account Information
- **Status:** Authenticated user with JWT token
- **Role:** "user" (in database)
- **Default Permissions:** Read products, manage cart, create orders

#### Characteristics
- Must be registered and logged in
- Has personal account with stored information
- Cart data persists across sessions
- Can view and manage own orders
- Cannot access admin/manager functions

#### Responsibilities
- Maintain account information accuracy
- Provide valid shipping address for orders
- Make timely payments (COD)
- Provide feedback through system

#### Key Actions
- ✅ Browse and search products
- ✅ Add/remove products from cart
- ✅ View cart and checkout
- ✅ Enter shipping information
- ✅ Complete payment
- ✅ View order history
- ✅ Track orders
- ✅ View order details and status

#### Primary Goals
- Purchase desired products
- Track orders
- Manage account

#### Key Use Cases
- UC01: View Product Catalog
- UC05: Add to Shopping Cart
- UC06: Manage Shopping Cart
- UC07: Checkout Process
- UC08: View Order History
- UC09: Track Order Status

---

### 2.3 Manager (Staff/Store Administrator)

**Definition:** An employee of Webbanquanao who manages store operations, inventory, and customer orders.

#### Account Information
- **Status:** Authenticated user with JWT token
- **Role:** "manager" (in database)
- **Access Level:** Extended permissions for store management

#### Characteristics
- Must be registered by Admin with manager role
- Has access to management dashboard
- Cannot create other accounts (Admin only)
- Can perform store operations
- Cannot modify user roles (Admin only)
- Full visibility of orders and inventory

#### Responsibilities
- Add new products to the catalog
- Update product information and images
- Manage inventory and stock levels
- Process and fulfill customer orders
- Update order status
- Monitor sales and revenue
- Generate reports
- Resolve customer issues related to orders

#### Key Actions
- ✅ Create, read, update, delete products
- ✅ Update product stock/inventory
- ✅ View all orders
- ✅ Update order status (processing, shipped, delivered)
- ✅ View sales reports
- ✅ View revenue analytics
- ✅ Manage product categories and brands
- ✅ View customer information (for orders)

#### Primary Goals
- Maintain accurate product database
- Fulfill customer orders efficiently
- Optimize inventory levels
- Generate revenue reports

#### Key Use Cases
- UC10: Product Management (CRUD)
- UC11: Inventory Management
- UC12: Order Management
- UC13: Revenue Reports
- UC14: View Dashboard

---

### 2.4 Admin (System Administrator)

**Definition:** An administrator with complete system control and highest level of permissions.

#### Account Information
- **Status:** Authenticated user with JWT token
- **Role:** "admin" (in database)
- **Access Level:** Full system control (unrestricted)

#### Characteristics
- Typically created during system setup
- Has all manager permissions
- Can manage user accounts and roles
- Can create/delete other manager and customer accounts
- Full access to all system features
- Responsible for system health and security

#### Responsibilities
- Manage all user accounts
- Assign and modify user roles
- Monitor system health
- Backup and recover data
- Enforce security policies
- Create manager accounts
- Delete problematic accounts
- Manage system configurations
- Handle escalated issues

#### Key Actions
- ✅ All manager actions
- ✅ Create manager accounts
- ✅ View all user accounts
- ✅ Delete user accounts
- ✅ Update user roles and permissions
- ✅ Access system logs
- ✅ Manage system settings
- ✅ Create backup/restore data
- ✅ Monitor API usage

#### Primary Goals
- Ensure system stability
- Maintain data integrity
- Manage user access
- Enforce security

#### Key Use Cases
- UC15: User Account Management
- UC16: Role Management
- UC17: System Configuration

## 4. Actor Interaction Flow

```
Guest
  ↓ [Browse, Search, Filter]
  ↓ [Register]
  ↓
Customer (User Role)
  ├─ [Shopping workflow] → Complete Order
  └─ [Request staff access] → Promoted to Manager
     ↓
     Manager (Manager Role)
       ├─ [Manage products & inventory]
       └─ [Promotions] → Admin
          ↓
          Admin (Admin Role)
             [Full system control]
```

## 5. Authorization & Access Control

### 5.1 Protected Routes

Routes protected by middleware `protect` and `admin`:
- **protect:** Requires valid JWT token (authenticated users only)
- **admin:** Requires both JWT token AND admin/manager role

### 5.2 Role-Based Access Control (RBAC)

```
Unprotected Routes: /api/users/login, /api/products (GET)
Protected Routes:   /api/users (POST, GET), /api/orders, /api/ads
Admin Routes:       /api/users (GET, DELETE, POST manager)
Manager Routes:     /api/products (POST, PUT, DELETE), /api/orders (POST)
```
