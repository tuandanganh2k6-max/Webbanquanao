# Chương 8: Yêu cầu chức năng và Phi chức năng

Vì hệ thống tập trung hoàn toàn vào việc Quản trị Banner quảng cáo trực tuyến (mang nặng yếu tố Media hình ảnh và liên kết mở rộng), các yêu cầu kỹ thuật phải được tối ưu mạnh mẽ trên khía cạnh truyền tải.

## 8.1 Yêu cầu Chức năng (Functional Requirements)
1. **Theo dõi dữ liệu vòng đời (Lifespan Tracker):** Hệ thống có chức năng đối chiếu giữa `End_Date` của bảng Hợp đồng và Giờ Đồng Hồ Thực Tế, qua đó tự động tô viền đổ bảng máu đỏ để hối thúc Admin nhận diện ra việc hợp đồng đó văng ra ngoài khoảng thời hạn cho phép.
2. **Quản trị Tài nguyên File hình (Image Resource Handling):** Hệ thống lập trình có khả năng bắt sóng sự kiện "Xóa hợp đồng / Xóa Banner" để gọi lệnh nã đạn lên máy chủ điện toán đám mây với mục đích phi tang gỡ bỏ hoàn toàn file ảnh gốc khỏi vùng không gian lưu trữ cứng (nhằm tiết kiệm chi phí băng thông vĩnh viễn), thay vì chỉ giấu ảnh đi ở DB.
3. **Cơ chế Điều hướng Mật (Hyperlink Target Redirect):** Mọi Banner quảng cáo phải tuân thủ nghiêm chèn thuộc tính code `target="_blank"` vào thể bao bọc rễ của nó. Qua đó, khi nhấn chuột để xem QC, trang của nhà quảng cáo sẽ được mở ra ở **một tab trình duyệt mới tinh**, tuyệt đối không được phép đá văng hay ghi đè lên tab website chủ sở hữu hiện tại để giữ độ níu kéo người xem (Bounce Rate).

## 8.2 Yêu cầu Phi chức năng (Non-Functional Requirements)
1. **Lá chắn chống tiêm (URL Injection Safe):** Vì hệ thống cấp quyền mở rộng dán Link URL bất kỳ từ đối tác. Yêu cầu bộ giải mã Back-end phải lọc chuỗi chắt lọc gắt gao (Regex Filtering), kiểm tra nếu Link quảng cáo bị nhập lén mã độc Javascript độc hại dạng chuỗi `javascript:alert('Hacked')` thì phải bắn tường lửa cản trả với mã Code lỗi 403.
2. **Hiệu suất tải tệp Đa truyền thông (Media Speed):** Sử dụng các vùng chứa (Container) giải mã nén đuôi File mạnh trước khi nhúng. File banner nặng 2MB phải được xử lý load vọt xuống dưới mức 300 mili-giây, nếu không sẽ làm trì trệ tốc độ tải trang chủ chung, gây khó chịu cho Khách.
3. **Responsive che phủ (Adaptive Layout Bounds):** Đảm bảo không xảy ra thảm họa đè lấp giao diện. Ở điểm đứt (breakpoint CSS) dưới độ phân giải máy Mobile (< 768px), những thanh đai quảng cáo này bắt buộc phải bị áp đặt luật `"tàng hình / khóa chìm"` để bù đắp diện tích chữ màn hình khiêm tốn của người cầm điện thoại xem di động.
