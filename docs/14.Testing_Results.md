# 🏆 Cài đặt & Kết quả Thử nghiệm Hệ thống Toàn diện

Tài liệu này trình bày các minh chứng về việc thiết lập môi trường thử nghiệm và kết quả đạt được sau khi triển khai các chức năng cốt lõi của Webbanquanao.

## 1. Cấu trúc Thử nghiệm
Hệ thống áp dụng mô hình kim tự tháp thử nghiệm (Testing Pyramid):
- **Unit Test:** Kiểm tra các hàm logic xử lý dữ liệu và Validation Model.
- **Acceptance Test:** Kiểm tra luồng nghiệp vụ của API thông qua các Request giả lập.
- **Behavior Driven Development (BDD):** Kiểm tra hành vi hệ thống dựa trên ngôn ngữ tự nhiên.

## 2. Kịch bản BDD (file.feature)
Chúng tôi sử dụng Gherkin để định nghĩa các Feature chính:
- `user_auth.feature`: Đăng ký, Đăng nhập.
- `shopping_cart.feature`: Thêm hàng và Thanh toán.
- `ad_management.feature`: Quản lý hợp đồng đối tác.
