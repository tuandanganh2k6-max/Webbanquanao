# Chương 9: Sơ đồ Biểu diễn Use Case (Use Case Diagram)

Sơ đồ bên dưới đây vẽ lại khối đồ họa luồng chức năng thao tác nghiệp vụ, qua đó hình dung được tầm kiểm soát bao phủ của nhân vật Quản Trị Viên đối với Phân hệ Hợp đồng mảng Truyền Thông Quảng Cáo.

```mermaid
usecaseDiagram
    actor "Quản trị viên (Admin)" as admin
    
    package "Advertisement & Contracts System CMS" {
        usecase "Quản lý Hồ sơ Đối tác" as UC1
        usecase "Thêm, Sửa Đối tác" as UC1_1
        usecase "Xóa Đối tác" as UC1_2
        
        usecase "Quản lý Hợp đồng Quảng cáo" as UC2
        usecase "Lập Hợp đồng mới" as UC2_1
        usecase "Báo cáo Thời hạn quá ngày" as UC2_2
        
        usecase "CMS Cấu Hình Banner" as UC3
        usecase "Upload & Lưu URL Banner" as UC3_1
        usecase "Gạt Bật/Tắt Hiển thị (Toggle Active)" as UC3_2
        
        usecase "Hệ điều phối xóa Cứng Dữ liệu Tự động" as UC4
    }
    
    admin --> UC1
    UC1 ..> UC1_1 : <<include>>
    UC1 ..> UC1_2 : <<include>>
    
    admin --> UC2
    UC2 ..> UC2_1 : <<include>>
    UC2 ..> UC2_2 : <<extend>>
    
    admin --> UC3
    UC3 ..> UC3_1 : <<include>>
    UC3 ..> UC3_2 : <<include>>
    
    UC1_2 ..> UC4 : Bấm Xóa Đối tác tự động kéo đứt chuỗi HĐ <<include>>
    UC4 ..> UC2 : Xóa chùm
    UC4 ..> UC3 : Dọn vùng lưu Media Storage
```

### Diễn giải Biểu đồ Kỹ thuật:
- **Nguyên lý gốc rễ:** Sơ đồ chỉ chấp nhận 1 Node Actor nguồn là `Admin`, đóng mọi cổng luồng tới tác nhân dân sự thường.
- Quan hệ `<<include>>`: Biểu thị sự cưỡng chế. Việc upload ảnh (UC3.1) và thiết lập url là điều bắt buộc phải có để tạo ra nghiệp vụ Cấu hình banner gốc (UC3). 
- Độc lập tự trị `UC4`: Lập biểu đồ kết chùm luồng xử lý xóa (cực hay trên thực tế CSDL Mongoose). Xóa đối tác (UC1.2) tự thân kích nổ khối C4 (luồng vô hinh ngầm), quét thẳng xóa tạch tạch những hợp đổng rác đi ké và giải phân bố RAM trống trên Cloud Ảnh.
