# Chương 5: Kịch bản BDD đặc tả Phân hệ Quảng Cáo

### 5.1 Thêm Hợp đồng Quảng Cáo
- **Happy Path (Thành công):**
  - `GIVEN` Admin đang ở trang Quản lý.
  - `WHEN` Admin điền đủ thông tin vào Form Thêm Hợp đồng và bấm Xác nhận.
  - `THEN` Hợp đồng được hệ thống ghi nhận và quảng cáo lập tức hiển thị ra màn hình Web.
- **Unhappy Path (Thất bại):**
  - `GIVEN` Admin đang điền Form Thêm Hợp đồng.
  - `WHEN` Admin điền sai thông tin (ví dụ cấm kỵ: Nhập Ngày kết thúc hợp đồng ở tận trong quá khứ).
  - `THEN` Hệ thống báo lỗi từ chối và cấm không cho lưu.

### 5.2 Tắt/Xóa Quảng Cáo
- **Happy Path (Thành công):**
  - `GIVEN` Đang có một quảng cáo hiển thị bám trên màn hình khách xem.
  - `WHEN` Admin không muốn mướn nữa nên ấn Công tắc tắt hoặc bấm nút Xóa vĩnh viễn hợp đồng đó ở Dashboard.
  - `THEN` Quảng cáo lập tức biến mất và bay màu ở ngoài giao diện Web.
- **Unhappy Path (Bắt lỗi):**
  - `GIVEN` Đang có một hợp đồng đã bị báo là "Quá hạn thời gian".
  - `WHEN` Admin buồn tay bấm "Bật" lại cái quảng cáo lố hạn đó lên.
  - `THEN` Hệ thống chặn lại không thèm hiển thị ra màn hình Web để tránh rác giao diện.
