# ✅ Tiêu Chí Chấp Nhận (Acceptance Criteria)

## Story 01-02: Browse & Search

### AC-Browse-001
**Story:** US01 - Xem Danh Sách Sản Phẩm  
**Tiêu Chí:**
- Trang /shop hiển thị grid sản phẩm 3 cột (responsive)
- Mỗi sản phẩm hiển thị: hình, tên, giá, \"Add to Cart\"
- Tải được tối đa 100 sản phẩm, paginate nếu > 12
- Load time < 3 giây
- Responsive: 320px (mobile), 768px (tablet), 1024px+ (desktop)

**Test Scenarios:**
1. ✓ Truy cập /shop → grid hiển thị
2. ✓ Scroll → load sản phẩm thêm (infinite scroll hoặc pagination)
3. ✓ Trên mobile → 1 cột, trên tablet → 2 cột, desktop → 3 cột

---

### AC-Search-002
**Story:** US02 - Tìm Kiếm Sản Phẩm  
**Tiêu Chí:**
- Search box ở header, input tìm keyword
- Tìm kiếm không phân biệt hoa/thường
- Khớp tên, mô tả sản phẩm
- Response time < 2 giây
- Hiển thị \"No products found\" nếu không match

**Test Scenarios:**
1. ✓ Tìm \"áo\" → hiển thị tất cả áo
2. ✓ Tìm \"THUN\" (viết hoa) → vẫn match \"thun\"
3. ✓ Tìm \"xyz\" (không tồn tại) → \"No products found\"

---

## Story 03-04: Authentication

### AC-Register-003
**Story:** US05 - Đăng Ký Tài Khoản  
**Tiêu Chí:**
- Form: Name, Email, Password (minimum 8 chars)
- Email format validation
- Email unique check (không cho phép duplicate)
- Password hashed bcryptjs (salt rounds 10)
- User role mặc định = \"user\"
- Success → redirect /login

**Test Scenarios:**
1. ✓ Đăng ký email mới, pass 8+ chars → success
2. ✓ Đăng ký email trùng → error \"Email đã tồn tại\"
3. ✓ Pass < 8 chars → error \"Mật khẩu tối thiểu 8 ký tự\"

---

### AC-Login-004
**Story:** US06 - Đăng Nhập  
**Tiêu Chí:**
- Email & password validation
- Compare password với bcrypt hash
- JWT token 30-day expiry
- Token lưu localStorage
- User info lưu trong state
- Redirect / sau login

**Test Scenarios:**
1. ✓ Email không tồn tại → error \"Email hoặc password sai\"
2. ✓ Password sai → error \"Email hoặc password sai\"
3. ✓ Email & password đúng → JWT tạo, redirect /

---

## Story 05-07: Shopping

### AC-Cart-005
**Story:** US07 - Thêm Vào Giỏ  
**Tiêu Chí:**
- Kiểm tra stock > 0
- Lưu: productId, quantity, size, color, price
- Cart key = productId + size + color
- Cập nhật cart counter
- Alert thành công \"Added to cart\"

**Test Scenarios:**
1. ✓ Chọn sản phẩm, qty 2 → thêm vào giỏ
2. ✓ Thêm cùng sản phẩm lần 2 → increment qty
3. ✓ Sản phẩm hết → \"Out of Stock\", disable nút

---

### AC-Checkout-006
**Story:** US10 - Thanh Toán Đơn Hàng  
**Tiêu Chí:**
- Form: name, email, phone, address, city
- Validate tất cả field
- Verify stock cho mỗi item
- Create Order document
- Reduce product stock
- Clear cart
- Send confirmation email
- Order ID sinh tự động

**Test Scenarios:**
1. ✓ Nhập đầy đủ address → order created
2. ✓ Quên address → error \"Vui lòng nhập địa chỉ\"
3. ✓ Stock = 0 → error \"Sản phẩm hết hàng\"

---

## Story 08-09: Management

### AC-ProductMgmt-007
**Story:** US13 - Tạo Sản Phẩm  
**Tiêu Chí:**
- Form: name, description, price, category, brand, stock, image
- Validate name unique
- Price > 0, stock >= 0
- Upload image, store file
- Product appears in catalog immediately
- Only manager/admin can create

**Test Scenarios:**
1. ✓ Điền đầy đủ → product created
2. ✓ Name trùng → error \"Tên sản phẩm đã tồn tại\"
3. ✓ Price < 0 → error \"Giá phải > 0\"

---

### AC-OrderStatus-008
**Story:** US18 - Cập Nhật Trạng Thái  
**Tiêu Chí:**
- Status flow: pending → processing → shipped → delivered
- Update Order document
- Send email to customer khi status change
- Email content tùy status (e.g., tracking info nếu shipped)
- Manager/admin only

**Test Scenarios:**
1. ✓ Order pending, change to shipped → email sent
2. ✓ Customer receives email với delivery estimate
3. ✓ Status history recorded
